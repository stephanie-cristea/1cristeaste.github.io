The Power of Shiny for Toronto Auto Theft Data

It was during another procrastination session on Reddit when I stumbled upon a post created by someone in the R programming subreddit asking for help programming a Shiny App.
Having never heard of Shiny, it sparked a flurry of google searches and tutorial binge watches and before I knew it, I was committing all my time to create interactive web applications.
Shiny is an open package created by RStudio for programmers to create interactive web applications. It requires no previous HTML, CSS or JavaScript background and provides advanced analytics to visualize even complex data in 3D plots and maps.
Using public data provided by the Toronto Police Service, I have created a visualizer for Toronto Auto Thefts between the years 2014 and 2018.
Inspired by the final project I worked on for my statistics course, I wanted to focus on demonstrating the evolution of the number of auto thefts depending on time. Our analyses showed that there was strong evidence to support a significant difference between thefts during the day vs during the night, but we also analyzed differences between months, all of which I wanted to include in the visualization.

Discussion

Shiny is a powerful tool for visualization. Further analysis would have to be conducted to discover any meaningful correlations between neighborhood(s) and/or month(s) for the number of auto thefts, however, using a visualizer such as this one would be an excellent way to develop possible hypotheses. These could, later on, be used to determine when and where police should be situated to prevent and minimize the number of auto thefts.
In addition, other factors may also need to be considered, however for my purposes, I have only included three factors.
I highly encourage anyone looking into developing their statistical and data science background to learn to program not only in R, but to also program in R with Shiny.
